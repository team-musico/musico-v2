# Musico v2 기획 문서

## 1. 프로젝트 개요

Musico v2는 YouTube iframe과 음원/lyrics 영상을 활용해 무료 음악 스트리밍 경험을 제공하는 웹 앱입니다. 사용자는 차트, 신곡, 검색, 관리자 편성 플레이리스트에서 곡을 고르고, 검증된 YouTube 영상만 큐에 추가해 재생할 수 있습니다.

디자인 방향은 실제 아이팟형 MP3 플레이어를 웹 레이아웃으로 구현하는 것입니다. 전체 페이지는 물리적인 플레이어 본체처럼 보이고, 내부 스크린 영역이 홈/차트/신곡/검색/플레이리스트/재생목록 페이지로 라우팅됩니다.

## 2. 핵심 가치

- 무료 접근성: YouTube iframe 기반으로 별도 음원 파일 없이 재생합니다.
- 검증된 검색: `{아티스트명} {노래제목} lyrics` 쿼리로 검색하고, 제목 포함 여부와 iframe 가능 여부를 검증합니다.
- 빠른 조작: 큐, 반복, 셔플, 이전/다음곡을 한 화면에서 처리합니다.
- 편성형 플레이리스트: 일반 사용자가 직접 생성하는 목록이 아니라 관리자가 팟캐스트처럼 공개 플레이리스트를 편성합니다.
- 차별화된 UI: 일반 스트리밍 앱의 미니멀 스타일이 아닌 물리적인 MP3 플레이어 같은 조작감을 제공합니다.

## 3. 기술 스택

- Framework: Next.js 16, App Router
- Language: TypeScript
- Styling: Tailwind CSS
- Auth: better-auth
- Backend/Data: Supabase
- Player: react-player
- YouTube Search: youtubei.js Innertube

## 4. 사용자 흐름

1. 사용자가 아티스트명과 노래 제목을 입력합니다.
2. 또는 사용자가 Discover 목록에 노출된 곡을 클릭합니다.
3. 서버 API가 `{artist} {title} lyrics` 쿼리로 Innertube 검색을 수행합니다.
4. 검색 결과 후보의 제목에 정규화된 아티스트명과 제목이 모두 포함되는지 확인합니다.
5. 후보 영상이 YouTube oEmbed 응답을 통해 iframe 재생 가능한지 확인합니다.
6. 검증을 통과한 첫 번째 영상을 큐에 추가하고 즉시 재생합니다.
7. 사용자는 재생, 일시정지, 이전/다음, 셔플, 전체 반복, 1곡 반복을 조작합니다.
8. 관리자는 `/admin`에서 플레이리스트를 만들고, 곡 검색 결과를 재생 가능한 영상으로 검증한 뒤 플레이리스트에 추가합니다.

## 5. 주요 기능

### 5.1 검색 및 영상 검증

- 입력값: `artist`, `title`
- 검색 쿼리: `${artist} ${title} lyrics`
- 검증 조건:
  - 후보 영상 제목에 아티스트명이 포함되어야 합니다.
  - 후보 영상 제목에 노래 제목이 포함되어야 합니다.
  - YouTube iframe/oEmbed 재생이 가능해야 합니다.
- 검증은 서버 Route Handler에서 수행하고, 실패 시 사용자에게 재생 가능한 음원을 찾지 못했다는 메시지를 표시합니다.

### 5.2 Discover 곡 목록

- 사용자가 앱에 들어오자마자 선택할 수 있는 추천곡 목록을 제공합니다.
- 곡 행에는 제목, 아티스트, 앨범, 분위기 태그, 재생 버튼을 표시합니다.
- 곡을 클릭하면 검색 폼 입력 없이 동일한 검증 API를 호출하고, 통과한 YouTube lyrics 영상을 플레이어에 즉시 로드합니다.
- 검증에 실패한 곡은 재생 가능한 음원을 찾지 못했다는 에러 메시지를 보여줍니다.

### 5.3 플레이어

- react-player의 YouTube 플레이어를 사용합니다.
- 현재 곡의 원본 YouTube 제목과 요청한 아티스트/곡명을 함께 표시합니다.
- YouTube iframe은 재생 엔진으로만 사용하고, 화면에는 자체 MP3 플레이어 UI를 표시합니다.
- App Router의 플레이어 레이아웃 안에서 내부 화면만 URL로 이동하므로 페이지 전환 중에도 음악이 끊기지 않습니다.

### 5.4 큐

- 검색한 곡을 큐에 추가합니다.
- Discover 목록에서 클릭한 곡도 큐에 추가합니다.
- 큐 항목 클릭 시 즉시 해당 곡으로 이동합니다.
- 큐에서 개별 곡 제거가 가능합니다.
- 다음곡/이전곡 이동을 지원합니다.

### 5.5 반복 및 셔플

- 셔플 재생을 지원합니다.
- 반복 모드는 `off`, `all`, `one` 세 가지입니다.
- `all`: 큐 마지막 곡 이후 첫 곡으로 이동합니다.
- `one`: 현재 곡을 다시 재생합니다.

### 5.6 플레이리스트

- 일반 사용자는 공개 편성 플레이리스트를 조회하고 전체를 큐로 불러올 수 있습니다.
- 일반 사용자 화면에서는 플레이리스트 생성/삭제/곡 추가 기능을 제공하지 않습니다.
- 관리자는 `/admin`에서 플레이리스트 생성/삭제, 곡 검색, 곡 추가/삭제를 수행합니다.
- 플레이리스트는 Supabase `musico_admin_playlists` 테이블에 저장됩니다.

### 5.7 관리자 편성 도구

- `musico_users.role = 'admin'`인 로그인 사용자만 접근 가능한 Route Handler를 사용합니다.
- 곡 추가 시 VIBE 검색 결과를 기반으로 `/api/search`를 호출해 iframe 재생 가능한 YouTube 영상 ID를 먼저 확보합니다.
- 중복 곡은 같은 플레이리스트에 다시 추가하지 않습니다.

## 6. 데이터 모델

### Track

- `id`: 앱 내부 트랙 ID
- `videoId`: YouTube video ID
- `title`: 사용자가 요청한 곡 제목
- `artist`: 사용자가 요청한 아티스트명
- `requestedTitle`: 원 요청 제목
- `thumbnail`: YouTube 썸네일
- `duration`: YouTube 영상 길이
- `url`: YouTube watch URL
- `sourceTitle`: 검색 결과의 실제 영상 제목

### AdminPlaylist

- `id`: 플레이리스트 ID
- `title`: 플레이리스트 제목
- `description`: 플레이리스트 설명
- `songs`: 관리자 편성 곡 목록
- `createdAt`: 생성 시각
- `updatedAt`: 수정 시각

## 7. 확장 계획

- 관리자 플레이리스트 곡 순서 변경
- 관리자 권한을 better-auth 세션과 역할 테이블로 확장
- YouTube 후보 재검증 캐싱
- 실패한 검색어에 대한 대체 검색 전략
- 플레이리스트 이름 변경과 설명 수정

## 8. 유의 사항

- YouTube 영상의 iframe 재생 가능 여부는 영상 소유자의 설정과 지역 제한에 따라 달라질 수 있습니다.
- Innertube 검색 결과는 YouTube 정책과 응답 구조 변화에 영향을 받을 수 있습니다.
- 앱은 음원 파일을 직접 저장하거나 배포하지 않고 YouTube iframe을 통해 재생합니다.
